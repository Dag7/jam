Conversation Context — How Agents Remember

Three mechanisms work together to maintain continuity across messages

Persistence Layer
Session ID Resume
In-memory map stores session IDs per agent. Claude Code uses --resume flag to continue server-side conversation. Lost on app restart.
voiceSessions.set(agentId, result.sessionId) // Next call: args.push('--resume', sessionId)
Claude Code only
Conversation JSONL
Every exchange recorded to daily files. Loaded on next command to rebuild context. Auto-pruned after 180 days.
~/.jam/agents/sue/conversations/ ├── 2026-04-03.jsonl └── 2026-04-04.jsonl
All runtimes — persistent
SOUL.md
Living personality file that evolves over time. Contains persona, traits, goals, learnings. Included in every system prompt.
~/.jam/agents/sue/SOUL.md
All runtimes — persistent
Matched Skills
Trigger-based .md files in skills/ directory. Auto-matched by keyword against command text. Injected when relevant.
~/.jam/agents/sue/skills/ └── react-patterns.md
All runtimes — persistent
Execution Paths
One-Shot Execute
Spawns a fresh process per command. Context must be explicitly injected via system prompt + session resume. Used for voice commands and chat.
runtime.execute(enrichedProfile, text, { sessionId, // for --resume cwd, env, onProgress, signal })
Voice & Chat commands
Interactive PTY
Agent process stays alive in a persistent terminal. Maintains its own internal state — no explicit context passing needed. Input written directly to PTY.
// Process is already running ptyManager.write(agentId, formatted + '\r') // Agent remembers everything internally
Terminal / Stage view
Message Queue
Per-agent sequential queue prevents race conditions. Each message fully processed before next starts. Context rebuilt fresh for every command.
enqueueCommand(agentId, text, 'text') → processQueue() // sequential lock → voiceCommand() // rebuilds context
System Prompt Composition (every execute call)
1 Agent identity — name, workspace, execution environment ~200 chars
2 SOUL.md — personality, traits, goals, learnings variable
3 Recent conversation history — last 20 entries (user + agent messages) ~6000 chars
4 Matched skills — trigger-matched .md files from skills/ directory variable
5 Skill system instructions — how to create and use skills ~800 chars
Max 12,000 characters — head (identity/soul) and tail (instructions) preserved on truncation
