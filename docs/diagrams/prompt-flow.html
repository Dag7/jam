Prompt Flow — User to Agent

Process Layer
1
User Input
containers/CommandBarContainer.tsx
User types in textarea and presses Enter. Text captured in React state, dispatched via submitCommand().
submitCommand() → sendTextCommand(textInput.trim(), attachments)
2
Optimistic UI + IPC Invoke
hooks/useOrchestrator.ts
Adds user message to Zustand store immediately (optimistic). Invokes IPC to main process.
await window.jam.chat.sendCommand(text, files)
3
Command Parsing & Routing
electron/ipc/chat-handlers.ts
Main process handler parses intent via CommandParser, resolves target agent via CommandRouter, checks for special commands (status, interrupt).
commandParser.parse(text) → router.resolveTarget() → router.dispatch()
4
Message Queue
packages/agent-runtime/src/agent-manager.ts
Per-agent queue with sequential processing lock. Messages processed one at a time to prevent race conditions.
agentManager.enqueueCommand(targetId, text, 'text') → processQueue() → voiceCommand(agentId, text)
5
Context Enrichment
packages/agent-runtime/src/agent-context-builder.ts
Builds system prompt from SOUL.md + last 20 conversation entries + matched skills. Max 12K chars.
contextBuilder.buildContext(profile, text) → { soulContent, recentHistory, matchedSkills } → enrichedProfile
6
Runtime Execute (Template Method)
packages/agent-runtime/src/runtimes/base-runtime.ts
Spawns child process with built args & env. Writes prompt to stdin. Captures output via JSONL or throttled strategy. Waits for exit.
spawnProcess(command, args, { cwd, env }) → writeInput(child, profile, text) // stdin write → outputStrategy.process(stdout) // parse response → ExecutionResult { text, sessionId }
7
Record & Respond
agent-manager.ts → agent-context-builder.ts
Saves user + agent messages to daily JSONL. Stores session ID for resume. Returns result to renderer for display.
recordConversation(cwd, { role: 'user', content: text }) recordConversation(cwd, { role: 'agent', content: result.text }) voiceSessions.set(agentId, result.sessionId)
