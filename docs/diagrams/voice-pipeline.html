Voice Pipeline — Speak to Agent and Back

Voice State capturing
Audio Capture
Browser MediaRecorder captures mic audio. Push-to-talk or voice activity detection. Audio chunks sent via IPC to main process.
Voice State processing
Speech-to-Text
Audio buffer sent to STT provider API. Returns transcription with confidence score and language detection. Agent name used as Whisper hint for accuracy.
Whisper (OpenAI) ElevenLabs Scribe
Routing dispatch
Command Parsing & Routing
CommandParser extracts intent + target agent from transcription. CommandRouter resolves agent ID by name match. Special commands (status, interrupt) dispatched directly.
Agent thinking
Agent Execution
Context enriched with SOUL.md + history + skills. Runtime spawns CLI process, writes prompt to stdin. Progress phrases sent to TTS during execution for real-time feedback.
Voice State speaking
Text-to-Speech
Agent response text sent to TTS provider. Audio cached by SHA256 hash. Queued playback via useTTSQueue hook — sequential, non-overlapping. Per-agent voice assignment.
OpenAI TTS (13 voices) ElevenLabs (48 voices)
Voice State idle
Record & Reset
Conversation recorded to daily JSONL. Session ID stored for resume. Voice state returns to idle, ready for next command.
